General Questions

Can I create challenges for HackerRank?

Sure, we are open to contributions! Just follow these guidelines and then you can submit it to us as at hackers@hackerrank.com. If you would like to become a paid contributor, please email us at hackers@hackerrank.com to apply.

What kind of challenges are we looking for?

All kinds! We are open to challenges in any domain that involves programming. We obviously take computer science challenges in topics like algorithms and artificial intelligence, but we are also interested in other subjects that involve programming, like physics, bioinformatics, and statistics.

Challenge Guidelines

Challenge Submissions

Standard challenges involve a series of test cases that the user needs to pass. We do not need any code from you to run such challenges. These are the things we need:

Input and Output Files

You will need to submit 5 -10 input and output files in increasing difficulty. Each input file should increase in size by a factor of 2 to 10 . The smaller input file should be solvable with a naive solution, while the larger ones should require an optimal solution. Your input files should also cover any edge cases of the problem.

If a challenge does not require a special optimal solution, you can submit fewer test cases. Just make sure to cover the edge cases.

Solution Code

You will also need to submit a solution to the code (in any language) so we can test out the challenge. The solution should have some comments to explain how it works. Also please send us a brief explanation that explains your approach to solving the problem and why it is correct.

Input Generator

Please also include the code you used to generate the test cases. We usually require this in case there is some issue with the test cases or to check what constraints were used.

Alternative Challenges

We also have other types of challenges on HackerRank:

CodeGolf Challenges - These challenges are graded based on the length of the source code. They can be created in the same way as standard challenges, but they need to be marked as “code golf”. Try to create challenges that would not be too difficult to solve with regular code, but are open to clever tricks to shorten code length.

Approximate Challenges - These challenges do not have exact solutions and do not require output cases. Instead, they require a code-checker to provide a score for each user. This does not yet have an interface for uploading, so it will need to be emailed to us.

Games - We don't currently accept games from problem-setters but let us know if you're interested in creating them.

Submission Dashboard

We will provide a dashboard for problem-setters to upload the challenge content and the input and output cases. Additional information, such as the solution code, can be emailed to us.

Content Guidelines

We created a template to outline what you need to cover in a standard challenge. The content should be in markdown and can be written on our online interface.

You should aim for extreme clarity in your challenge description, so users can understand the challenge quickly and start solving it immediately. Often, diagrams get a point across faster than paragraphs, so please include them when possible.

The main part of the challenge is the formal description, but it should also have some kind of story introduction which explains how such a challenge could come up. While these scenarios do not need to be realistic, try to find something realistic if possible.

Payment for creating challenges

Easy Challenge
An easy challenge involves a straightforward concept to solve a problem and can usually be solved quickly by an experienced user. An example of this would be 2's complement. We pay $50 for every challenge of this type.

Medium challenge
A medium challenge requires analysis to figure out the correct solution, but an experienced user will not have trouble solving it. An example of this would be Zombie March. We pay $100 for every challenge of this type.

Hard Challenge
The solution to a hard challenge will not be direct and will require special insight to find. Even an experienced user will need to struggle to find a solution. An example of this would be Queens Revisited. We pay $150 for every challenge of this type.

Approximate algorithm/Big-data & ML challenges
These are challenges where there's not a clear solution to the problem and users can keep optimizing their algorithms forever. These challenges might also involve machine learning concepts. We pay anywhere between $100-$200 for this challenge.
